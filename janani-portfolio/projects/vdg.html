Revocable identity based proxy re-encryption for data sharing in clouds

  • Tech Stack: Dotnet, SQL, javascript, bootstrap, html, css

Revocable identity-based broadcast proxy re-encryption (RIBBPRE) is a cryptographie technique that enables secure and efficient data sharing in cloud computing environments. In this scheme, data owners can delegate access rights to multiple users by encrypting the data once and then using proxy re-encryption to generate ciphertexts that can be decrypted by the authorized users. The RIBBPRE scheme is based on identity-based encryption (IBE), which allows users to use their identity, such as their email address, as their public key. This eliminates the need for a public key infrastructure (PKI), which can be complex and difficult to manage in large-scale systems. In addition to IBE, the RIBBPRE scheme uses proxy re- encryption, which allows a third party, known as a proxy, to transform ciphertexts encrypted under one user's public key into ciphertexts that can be decrypted by another user. This enables data owners to delegate access rights to multiple users without having to re-encrypt the data multiple times. The revocability feature of the RIBBPRE scheme allows data owners to revoke access rights to specific users without having to change the encryption keys or re-encrypt the data. This is achieved by using a revocation list that is maintained by the data owner and the proxy. When a user is revoked, the proxy simply removes the user's decryption capability from the revocation list. Overall, the RIBBPRE scheme provides a flexible and efficient solution for secure data sharing in cloud computing environments, particularly in scenarios where access rights may need to be revoked or updated frequently.
